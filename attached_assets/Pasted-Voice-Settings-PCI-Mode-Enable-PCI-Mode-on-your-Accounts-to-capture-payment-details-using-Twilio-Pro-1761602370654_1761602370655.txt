PCI Mode
Enable PCI Mode on your Accounts to capture payment details using Twilio Programmable Voice. PCI Mode is required for customers that create PCI workflows with Programmable Voice. Enabling PCI Mode on your Account redacts sensitive payment details captured using Twilio Programmable Voice and <Pay>. PCI Mode will automatically store call recordings when recording is enabled on your PCI workflow. To transcribe voice recordings, customers must use the Voice <Transcription> noun, Native and Marketplace transcriptions are not available when PCI Mode is enabled. For more information regarding PCI Mode and Recordings retention policy, please refer to applicable product documentation here.

Once PCI Mode is enabled, it cannot be disabled for that Account. Please see Twilio's Responsibility Matrix and Programmable Voice Documentation to learn more about your obligations when using Programmable Voice in a PCI workflow.

Request Inspector
Request Inspector is a tool to debug call flows. It logs the HTTP requests and responses made during the call, including headers, parameters, and body. These logs are stored for 30 days after the call is created. It is recommended to be Enabled when debugging and troubleshooting your TwiML applications.

When Disabled, the Request Inspector does not log any response bodies, and is recommended when you have data compliance requirements around sensitive personal information. Disabling this tool does not delete existing log entries.


Request Inspector
Enabled

Disabled
When disabled, the Request Inspector does not log any response bodies. Recommended when you have data compliance requirements around sensitive personal information. Disabling this tool does not delete existing log entries.
HTTP Basic Authentication for media access
In order to protect media access stored at Twilio, you can enforce authentication to access them by enabling HTTP Basic Authentication in your Twilio Account. This setting requires your Twilio Account SID and Auth Token or API Key for all requests to access Voice and Messaging media files, such as recordings and messages' images. Twilio highly recommends enabling HTTP Basic Authentication for your media, especially if it contains sensitive data. Learn more


HTTP Basic Authentication for media access
Enabled
Media is protected from public access and HTTP Basic Authentication is required to access media urls.
Disabled
Media is not protected from public access and anyone with the media url can access it.

If enabled, your SIP endpoints can use the deprecated TLS 1.0 and 1.1 versions for SIP signaling sent to or received from Twilio. If disabled, only non-deprecated versions (TLS 1.2+) are allowed. Attempts to initiate calls over less secure connections will fail. It is highly recommended to disable this setting unless required by your SIP endpoints. Learn more
